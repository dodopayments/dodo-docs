## Overview

The Dodo Payments License Key Management system provides comprehensive APIs for creating, validating, and managing software licenses. This system is ideal for software vendors who need to control access to their digital products through license key validation.

## Key Concepts

<AccordionGroup>
  <Accordion title="License Keys">
    Unique identifiers that grant access to your software or digital products. Each license key can have:
    - Activation limits (number of devices/installations)
    - Expiration dates
    - Product-specific access rights
    - Custom metadata for additional features
  </Accordion>
  
  <Accordion title="License Instances">
    Individual activations of a license key on specific devices or installations. Each instance tracks:
    - Device information and fingerprint
    - Activation timestamp
    - Last validation check
    - Active/inactive status
  </Accordion>
  
  <Accordion title="Validation Process">
    Real-time verification that ensures:
    - License key authenticity
    - Activation limit compliance
    - Expiration date validity
    - Product access permissions
  </Accordion>
</AccordionGroup>

  <Step title="Basic License Workflow">


## Code Examples

<Tabs>
  <Tab title="License Activation">
    ```javascript license-activation.js icon="js" expandable
    // Example: Activate a license key
    const activateLicense = async (licenseKey, deviceInfo) => {
      try {
        const response = await fetch('/api/licenses/activate', {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
          },
          body: JSON.stringify({
            license_key: licenseKey,
            device_name: deviceInfo.name,
            device_fingerprint: deviceInfo.fingerprint,
            metadata: {
              os: deviceInfo.os,
              version: deviceInfo.version
            }
          })
        });
        
        const result = await response.json();
        
        if (response.ok) {
          // License activated successfully
          localStorage.setItem('license_instance_id', result.instance_id);
          return { success: true, instance: result };
        } else {
          // Handle activation errors
          return { success: false, error: result.message };
        }
      } catch (error) {
        return { success: false, error: 'Network error during activation' };
      }
    };
    ```
  </Tab>
  
  <Tab title="License Validation">
    ```javascript license-validation.js icon="js" expandable
    // Example: Validate an active license
    const validateLicense = async (licenseKey) => {
      try {
        const response = await fetch('/api/licenses/validate', {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
          },
          body: JSON.stringify({
            license_key: licenseKey,
            device_fingerprint: getDeviceFingerprint()
          })
        });
        
        const result = await response.json();
        
        if (response.ok && result.valid) {
          // Update local cache with validation result
          localStorage.setItem('last_validation', Date.now());
          return { valid: true, expires_at: result.expires_at };
        } else {
          return { valid: false, reason: result.reason };
        }
      } catch (error) {
        // Handle network errors - check cached validation
        const lastValidation = localStorage.getItem('last_validation');
        const gracePeriod = 7 * 24 * 60 * 60 * 1000; // 7 days
        
        if (lastValidation && (Date.now() - lastValidation) < gracePeriod) {
          return { valid: true, cached: true };
        }
        
        return { valid: false, reason: 'Unable to validate license' };
      }
    };
    ```
  </Tab>
  
  <Tab title="Device Management">
    ```javascript device-management.js icon="js" expandable
    // Example: Manage device activations
    const manageDevices = {
      async listDevices(licenseKey) {
        const response = await fetch(`/api/licenses/${licenseKey}/instances`, {
          headers: { 'Authorization': `Bearer ${apiKey}` }
        });
        return await response.json();
      },
      
      async deactivateDevice(instanceId) {
        const response = await fetch('/api/licenses/deactivate', {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
          },
          body: JSON.stringify({ instance_id: instanceId })
        });
        return await response.json();
      },
      
      async transferLicense(oldInstanceId, newDeviceInfo) {
        // Deactivate old device
        await this.deactivateDevice(oldInstanceId);
        
        // Activate on new device
        return await activateLicense(licenseKey, newDeviceInfo);
      }
    };
    ```
  </Tab>
</Tabs>

## Best Practices

<AccordionGroup>
  <Accordion title="Security Considerations">
    - **Key Protection**: Never hardcode API keys in client applications
    - **Device Fingerprinting**: Use reliable, privacy-compliant device identification
    - **Validation Frequency**: Balance security with user experience
    - **Error Handling**: Provide clear, actionable error messages to users
  </Accordion>
  
  <Accordion title="User Experience">
    - **Seamless Activation**: Minimize steps in the activation process
    - **Offline Support**: Implement graceful degradation for network issues
    - **Device Management**: Allow users to manage their activated devices
    - **Clear Messaging**: Explain license limits and expiration clearly
  </Accordion>
  
  <Accordion title="Operational Excellence">
    - **Monitoring**: Track license usage and validation patterns
    - **Analytics**: Monitor activation success rates and common issues
    - **Support**: Provide tools for customer support to help with license issues
    - **Compliance**: Ensure license validation respects user privacy
  </Accordion>
</AccordionGroup>
